Soma GitHub

Native iOS prototype · Web showcase

A clearer way to describe what you feel.

Soma helps you structure a physical symptom before speaking to a healthcare professional. Tap where it hurts on a calm, neutral body — and build a picture you can actually share.

What you see below are real screenshots of the native SwiftUI app running in the iPhone simulator. The app itself runs on iOS — this page is only a showcase.

Product principles

  • Calm. A quiet, atmospheric scene instead of a clinical form.
  • Private. The prototype keeps every check-in on the device — no accounts, no network.
  • Precise. One shared coordinate system keeps the body and its regions aligned on every screen size.
  • Honest. Soma structures what you feel. It never guesses what it means.

What the prototype includes

Current scope

The first step of the flow: choosing where it hurts. Back view by default, front/back switching, a selectable lower-back region with haptic feedback, and navigation into a placeholder for the next step. Later steps — refining the area, severity, sensation, and a doctor-ready summary — are not built yet.

Accessibility

VoiceOver labels, values, and selection states; a named action to select the region without precise tapping; Dynamic Type up to accessibility sizes; Reduce Motion variants for every animation; 44-point minimum targets; state is never communicated by colour alone.

Under the hood

Native SwiftUI, no external dependencies. The figure is a single mirrored Bézier outline rendered with layered light in a Canvas. State logic is covered by Swift Testing unit tests and built in CI.

Disclaimer

Soma is an early interface prototype. It does not provide diagnoses or medical treatment.
